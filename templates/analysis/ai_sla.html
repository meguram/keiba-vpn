実行方針： スクレイパーの定期 cron が完了したタイミングを起点としてモデル推論を連鎖実行します。 トリガー A は「前日18時の出馬表・調教データ取得完了後」、 トリガー B は「当日レース15分前のオッズ取得完了後」に発火します。 推論結果は レース詳細ページ の「🤖 AI予測」セクションに反映されます。
Trigger A
前日 18:00 — 出馬表・調教データ取得完了後 race_shutuba / race_shutuba_past / race_oikiri が揃った時点でトリガー
実行フロー
前日18時データ取得 → 位置取り予測 + ペース予測 + 上り3F予測 → 追走難度分析 → /race 追走難度
前日18時データ取得 → 単複オッズ予測 → /race 想定オッズ
📍
位置取り予測
Trigger A
各馬がレースで採る想定ポジション（逃・先行・差し・追込）の確率分布を推定します。ペース・コース形態・近走の脚質傾向を考慮します。
入力race_shutuba / race_shutuba_past / race_index
出力馬ごとの位置取り確率分布
利用先追走難度分析（中間モデル）
⚡
ペース予測
Trigger A
LightGBM 回帰（1F / 3F の2ヘッド）でレース前半ラップタイムを予測します。逃げ頭数・コース・馬場・頭数などをフィーチャーとし、2022–2025 の 5,035 レースで学習しました。
入力race_shutuba / race_shutuba_past / race_index
出力前半1F・3F ラップタイム推定（秒）
登録先pace-lap-time-predictor-1f/3f (MLflow Registry, 1F MAE=0.191s)
利用先追走難度分析（中間モデル）
🏃
上り3F予測
Trigger A
各馬の上り3ハロンタイムを予測します。近走の上り実績・馬場・距離適性・脚質を特徴量として使用します。
入力race_shutuba / race_shutuba_past / race_index
出力馬ごとの上り3F推定タイム
利用先追走難度分析（中間モデル）

単複オッズ予測
Trigger A
LightGBM 3ヘッド（単勝 / 複勝下限 / 複勝上限）でオッズを回帰予測します。前日18時時点のデータのみを使用し、データリークを排除しています。
入力race_shutuba / race_shutuba_past / race_oikiri / race_index（前日18時時点）
出力単勝オッズ・複勝下限・複勝上限（馬ごと）
登録先keiba-final-odds-lgbm (MLflow Registry)
利用先/race 🤖 AI予測「想定オッズ」
Trigger B
当日レース 15分前 — オッズ取得完了後 race_odds / race_shutuba (fallback) の取得完了時点でトリガー
実行フロー
当日15分前データ取得 → 勝率 / 連対率 / 複勝率予測 → /race 予測確率・印
🏆
勝率 / 連対率 / 複勝率予測
Trigger B
