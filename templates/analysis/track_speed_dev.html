グルーピング → β算出 → 補正

# コンテキストキー（ベースライン構築時に計算） key = venue|layout|surface|dist|class|cond_pool # ペース指標を z 変換（Var=1 に正規化） z_pace = (metric_val − μ_pace) ÷ σ_pace # OLS回帰係数（Var(z_pace)=1 なので簡略化） β = Cov(z_pace, time_2nd) # 補正量（±1.5秒クリップ） adj = clip(β × z_pace, −1.5, +1.5) time_adj = time_2nd − adj # ペースラベル（指標値が大 = 前半遅い = スロー） z_pace < −1 → "前傾" (ハイペース) z_pace > +1 → "抑え" (スロー)

サンプル A · 標準ペース

府中S · μ=36.2 σ=1.0
z_pace = (36.3−36.2)÷1.0+0.08 (標準)
adj = β×0.08+0.03 秒（微小）
time_adj = 117.8 − 0.03≈ 117.8 秒
標準ペース → 補正量は無視できる水準

サンプル B · 極端なハイペース

摩周湖特別 · μ=35.2 σ=1.6
z_pace = (28.9−35.2)÷1.6−3.99 (前傾)
adj = clip(β×(−4.0), ±1.5)−1.5 秒（上限クリップ）

PF指数の定義

# 馬場速度との差分（1pt = 0.1秒） Δt = (pooled_mean_z − race_z) × baseline_std × 10 # コース内 PF指数（50 = 当日馬場速度と同等） perf_index = 50 + Δt # 全コース横断比較（東京芝2000m良 基準） scale = σ_2000m_良 ÷ σ_actual # [0.33, 3.0] clip perf_index_std2000 = 50 + Δt × scale
std2000 補正の意味
短距離（芝1000m）はσ小 → Δtを拡大して他コースと比較可
重馬場はσ大 → Δtを縮小してレベル差を公平評価

速度水準ラベル（クラス換算）

# 定義: PF=50に最も近いクラス = argmin σ距離 score_C = |(time_2nd_adj − mean_C) ÷ std_C| + 1÷√n_C class_level_label = argmin_C score_C # 制約 候補: |rank_C − own_rank| ≤ 2 # ±2クラス以内 条件: n_C ≥ 20 # 小サンプル排除
第2項 1/√n は平均推定誤差(SE/std)のペナルティ。n→∞ で消滅。
±2制約はベースライン逆転・G級データ不足による遠距離ジャンプを防ぐ。

📝 設計メモ・制約

ペース補正の限界 補正量は ±1.5秒でクリップ（過補正防止）
コンテキスト n < 30 はフォールバック（distance_band 単位）
RPCI は laps データがないと計算不可 → first_half_3f へ
テンポラルプーリング適用範囲 query_day のみ適用（query_by_category は各日単独）
馬場状態の最頻値が不一致なら前日・前週は除外
PF指数は assign_races 時に計算・parquet 保存で不変
速度水準ラベルの制約 ±2ランク制約: 全コースの73.8%でG級ベースラインが不在、34箇所で高クラスほど平均タイムが遅い逆転現象あり。無制限検索では2勝→G1の誤分類が5%発生
MIN_CLASSIFY_N=20: n=17等の不安定な平均に引きつけられるのを防ぐ
class_level_label はクエリ時に動的計算（parquet に保存しない）
std2000 補正テーブル σ_2000m良 は baselines.parquet から surface 別に計算
スケール係数は [0.33, 3.0] にクリップ（極端距離対策）
distance=0 のレースはスケール=1.0
baseline_std CAP baseline_std は 3.0秒でキャップ（BASELINE_STD_CAP）
distance=0 集約行の std が 20〜47 になる問題への対処
通常コース（距離既知）の std 範囲: 0.55〜2.88秒
