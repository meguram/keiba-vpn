Race Performance Rating System

レースパフォーマンス算出システム — 完全版ドキュメント

異なるレース間で比較できる、能力・条件・相手関係をならした指標をロジックベースで算出するシステムの設計と実装です。 2025 年データでの実測ベンチマーク、当日・週次の運用ルールもあわせてまとめています。

future_horizon_days = 0 race_id 単位蓄積 online-safe / retro-only 分離 LightGBM top1 hit ≈ 23.24% vs random 約 3.0×

01 目的

過去レースごとに、各馬の「そのレースでのパフォーマンス値」とレース自体の「レースレベル」を、 機械学習モデルの前段で使えるロジックベースの指標として算出します。

単純な着順の置き換えではなく、異なるレース間で比較できる、能力・条件・相手関係をならした指標を作ることが目的です。次のような問題意識があります。

  • 同じ2着でも、強い相手との2着は高く評価すべき
  • 同じ1.0秒差でも、短距離と長距離では意味が異なる
  • 勝ち馬の着差は「流して勝った」可能性があるため、そのまま能力差とは見なさない

02 基本方針

方針内容主な指標 / 備考
① 指標を分離する 能力・レベル・馬場・展開を独立した指標として算出する RunPerformance（馬の強さ）、RaceLevel（メンバー水準）、TrackVariant（時計の出やすさ）、PaceDifficulty（展開有利不利）
② 使える時点を分ける 本番予測用（将来情報なし）と過去分析用（後続戦績反映）を分離する RaceLevel_PreRace（online-safe）/ RaceLevel_Retrospective（retro-only、ラベル品質改善専用）
③ 着順だけに頼らない RunPerformance を 4 ブロックの合成で算出する TimeFigure MarginAdjustment WeightAdjustment RaceLevelAdjustment

運用上の重要ポイント

RaceLevel_Retrospective は将来情報を含むため、 予測の特徴量には使わず、過去レースの教師信号・ラベル品質の改善にのみ使うことが原則です。 これにより本番予測（online-safe）と過去分析（retrospective）を両立します。

05 TimeFigure（走破タイムをもとにした能力値）

層別キー（条件の組み合わせ）

venue_code × surface × distance_bucket × class_group × track_condition × venue_era

ベースタイムの求め方

  • 1〜3着馬の平均タイムを基本とする（極端な外れレースは除外）
  • 勝ち馬単独では「流し勝ち」の影響が出る。2着単独では展開の影響が出る。両方を避けるため平均を使う

計算イメージ

time_figure_base = scale(distance_bucket) * (baseline_time - adjusted_finish_time)

# adjusted_finish_time は track_variant / 斤量 / 当日馬場差を補正した後の走破時計

06 MarginAdjustment（着差の補正）

距離帯別「1ポイント = 何秒か」

帯距離 (m)
sprint～1400
mile1400-1800
intermediate1800-2200
long2200-2800
extended2800+

勝ち馬の緩め補正

winner: alpha * log1p(margin_seconds / sec_per_point)
others: margin_seconds / sec_per_point

勝ち馬は log1p で圧縮、2着以下は線形で計算します。

その他の考慮事項：ハナ・アタマ・クビの秒換算、同着、落馬・競走中止・失格の扱い。

weight_adjustment_seconds = (normalized_weight - reference_weight_for_race) * kg_to_seconds(distance_band)

標準斤量基準への正規化（実装済み）

過去レースの run_performance は標準斤量基準で保持し、次走で使う horse_pre_rating_pre_race は今回の出走斤量に応じて補正します。 これにより、軽斤量戦での過大評価リスクを抑えます。

08 TrackVariant（馬場差）

条件別ベースタイムと1〜3着平均、馬の能力偏差を反復推定して日次の馬場差を算出します。

retrospective 版（過去分析用）

全期間データで baseline と variant を再構成します。データ分析や教師ラベルの精緻化に使います。

online 版（本番予測用・実装済み）

過去日付のデータだけで baseline と variant を生成する track_variant_online を保持しています。週次・当日の特徴量にはこちらを使います。

  • 同日・同場・同サーフェスの前半レースで当日内のドリフトを補正する
  • クッション値・含水率は事前推定（prior）として活用する
  • track_condition はカテゴリ値、track_variant は連続値として両方保持する

09 PaceShapeAdjustment（展開による補正）

同じ走破タイム・着差でも、展開によって能力評価の意味が変わります。ペース・展開を分類して補正します。

展開分類特徴補正の考え方
消耗戦 ハイペース・持久戦 先行して粘った馬を加点する
持続戦 中間的な展開 フラットに評価する
瞬発戦 スロー上がり戦 後方から届かなかった馬を過剰に減点しない

10 RaceLevel（相手レベル）

A. race_level_pre_race（本番予測に使用可）

レース時点までの情報だけで算出します。

  • 出走馬の直近 run_performance_final
  • キャリア勝率・複勝率
  • 同条件成績、獲得賞金、クラス昇降履歴
  • 騎手・厩舎の OOF による強さスコア
race_level_pre_race
 = 0.6 * mean(top3 entrant_rating)
 + 0.4 * mean(all  entrant_rating)

B. race_level_retrospective（過去分析専用）

後続戦績まで反映して安定化させた値です。

  • レース後 180 / 365 / 730 日の獲得賞金
  • 重賞出走・好走
  • 後続レースの run_performance_final
  • 勝ち上がり頭数

将来情報を含む → 学習時の特徴量には使わない。

11 RunPerformance の合成式

run_performance_raw =
    a * time_figure
  + b * margin_adjustment
  + c * weight_adjustment
  + d * pace_adjustment
  + e * race_level_adjustment
  + f * trouble_adjustment
  • race_level_adjustment：弱いメンバーへの2着は減点、強いメンバーへの2着は加点
  • trouble_adjustment：出遅れ・不利・前壁・直線詰まりなどのロスを補正

12 追加の考慮事項

考慮事項内容使用指標 / 手法
不利・ロスの補正 出遅れ・前壁・詰まりなど展開ロスを補正する 通過順・上がりタイム・枠順から代理変数を作成
少頭数・多頭数 頭数によって評価の信頼性が変わる field_size_factor で race_uncertainty を調整
新馬・未勝利戦 戦績が少なく評価が不安定 race_uncertainty を高くし、後続戦績で retrospective 安定化
年齢・成長曲線 2歳・3歳春は能力変動が大きい retrospective 寄りの補正をやや強める
距離変更・転戦 距離帯変更や芝ダート転換時の不確実性 距離帯変更ペナルティ・転換時の不確実性考慮
当日の位置取り有利不利 展開の不利を受けた好走を正当評価する day bias を使って不利な展開での好走に加点

16 結果の解釈と改善ポイント

まとめ

run_performance 系の履歴だけで、2025年の race-wise top1 的中率を 23% 前後まで向上させました（ランダム基準 7.66% に対して約 3.0 倍）。 future_horizon_days = 0（将来情報なし）の本番想定ロジックのみで達成しており、 単独で有意な予測信号として機能しています。

重要な特徴量（is_win 上位）

margin_fig_avg5 perf_pct_last1 pace_fig_avg5 field_top3_gap pace_same_distance_avg5 margin_same_distance_avg5 perf_same_class_avg5 n_prev_runs perf_same_venue_avg5 perf_same_track_cond_avg5

改善の経緯

#改善内容状態
1distance / surface / field_size の上流補完完了
2track_variant の online 化完了
3race_level_pre_race の race 内差分化（field_relative_pre_rating / field_top3_gap）完了
4標準斤量基準への正規化と今回斤量補正完了
5条件帯標準化（_std / _pct）完了
6文脈別履歴集約（same_class / same_venue / same_track_cond / same_surface_distance）完了
7pace_same_distance_avg5 / margin_same_distance_avg5 の追加完了
8race store を source of truth 化（--race-ids）完了
9future_horizon_days = 0 を運用デフォルト化完了

改善の頭打ちについて

改善幅は十分に小さくなり、現行スコープでの大きな改善余地は出尽くした状態です。 今後の改善は、このロジック単体ではなく、次のような上流・下流との接続に移行します。

  • horse_pre_race_dataset への安全な統合
  • pace_shape_class 別の履歴集計
  • 多段階処理の流れとの接続

18 評価プロトコル

  1. 2020-2024 を履歴構築・学習期間とする
  2. 2025 を holdout（最終評価データ）とする
  3. 2025 年の指標は future_horizon_days=0 で再算出したものを使う
  4. race_level_retrospective は holdout 時の予測特徴量に含めない
  5. 本番想定での寄与評価は、常に online_safe の列だけで行う

19 完全版と判断できる根拠

future_horizon_days = 0 の本番想定ロジックで算出できる race_id 単位 での蓄積が可能 2025年 holdout で 有意な予測寄与を確認済み 特徴量として data/features に保存済み 再現可能な評価スクリプトを保持している

まとめ

現時点の race performance システムは、現行スコープでの完全版とみなせます。 今後はロジック単体の改善ではなく、上流・下流との接続（dataset 統合 / pace_shape 履歴 / 多段階処理の流れ）に進むのが適切です。
