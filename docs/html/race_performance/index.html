Race Performance Rating System

レースパフォーマンス算出システム — 完全版ドキュメント

異なるレース間で比較可能な、能力・条件・相手関係をならした指標を ロジックベースで算出する仕組みの設計と実装、2025 年データでの実測ベンチマーク、 当日 / 週次の運用ルールまでを一枚にまとめたリファレンスです。

future_horizon_days = 0 race_id 単位蓄積 online-safe / retro-only 分離 LightGBM top1 hit ≈ 23.24% vs random 約 3.0×

01 目的

過去レースごとに、各馬の そのレースでのパフォーマンス値 と、レース自体の レースレベル を、機械学習モデルの前段で利用できるロジックベースの指標として算出します。

単純な着順の置き換えではなく、異なるレース間で比較可能な、能力・条件・相手関係をならした指標 を作ることが目的です。

  • 同じ 2着 でも、相手が強いレースの 2着は高く評価したい
  • 同じ 1.0秒差 でも、短距離と長距離では意味が違う
  • 勝ち馬の 着差 は、流して勝っている可能性があるためそのまま能力差と見なさない

02 基本方針

① 指標を分離する

  • RunPerformance 馬のそのレースでの強さ
  • RaceLevel メンバー水準
  • TrackVariant 馬場・時計の出やすさ
  • PaceDifficulty 展開の歪み

② 時点を分ける

  • RaceLevel_PreRace — 将来リーク無し
  • RaceLevel_Retrospective — 後続戦績まで反映、教師信号精緻化用

予測特徴量と過去ラベル品質向上の両用途に対応。

③ 着順だけに頼らない

最終 RunPerformance は次の 4 ブロックを合成します。

TimeFigure MarginAdjustment WeightAdjustment RaceLevelAdjustment

運用上の重要ポイント

RaceLevel_Retrospective は future-aware なため、 未来予測の特徴量には使わず、過去レースの教師信号 / ラベル品質改善にのみ使う。 これにより online-safe 運用と retrospective 分析を両立します。

05 TimeFigure（走破時計ベース能力値）

層別キー

venue_code × surface × distance_bucket × class_group × track_condition × venue_era

ベースタイムの作り方

  • 1-3 着平均を基本（極端な外れレースは除外）
  • 勝ち馬単独 = 流し勝ち影響、2着単独 = 展開影響を回避

定義イメージ

time_figure_base = scale(distance_bucket) * (baseline_time - adjusted_finish_time)

# adjusted_finish_time は track_variant / 斤量 / 当日馬場差を補正した後の走破時計

06 MarginAdjustment（着差補正）

距離帯別「1ポイント = 何秒」

帯距離 (m)
sprint～1400
mile1400-1800
intermediate1800-2200
long2200-2800
extended2800+

勝ち馬の緩め補正

winner: alpha * log1p(margin_seconds / sec_per_point)
others: margin_seconds / sec_per_point

勝ち馬は log1p 圧縮、2着以下は線形。

追加考慮: ハナ／アタマ／クビの秒換算、同着、落馬・競走中止・失格の扱い。

weight_adjustment_seconds = (normalized_weight - reference_weight_for_race) * kg_to_seconds(distance_band)

標準斤量基準への正規化（実装済み）

過去レースの run_performance は標準斤量基準で正本保持し、次走で使う horse_pre_rating_pre_race は今回出走斤量に応じて補正。これにより軽斤量戦での過大評価リスクを抑制。

08 TrackVariant（馬場差）

条件別ベースタイムと 1-3 着平均、馬の能力偏差の反復推定で日次馬場差を生成。

retrospective 版

全期間データで baseline / variant を再構成。EDA や教師ラベル精緻化向け。

online 版（実装済み）

過去日付のみで baseline と variant を生成する track_variant_online を保持。週次・当日近似の特徴量にはこちらを使用。

  • 同日同場同 surface の前半レースで intraday drift を補正
  • クッション値・含水率は prior として利用
  • track_condition はカテゴリ、track_variant は連続量で両方保持

09 PaceShapeAdjustment（展開補正）

同じ時計・着差でも、展開によって能力評価の意味が変わるため、ペース・展開を分類して補正します。

消耗

ハイペース／持久戦。先行粘りを加点。

持続

中間的展開。フラットに評価。

瞬発

スロー上がり戦。後方届かずを過剰減点しない。

10 RaceLevel（相手レベル）

A. race_level_pre_race

レース時点までの情報で算出。

  • 出走馬の直近 run_performance_final
  • キャリア勝率 / 複勝率
  • 同条件成績、獲得賞金、クラス昇降履歴
  • 騎手・厩舎の OOF 強さ
race_level_pre_race
 = 0.6 * mean(top3 entrant_rating)
 + 0.4 * mean(all  entrant_rating)

B. race_level_retrospective

後続戦績まで見て安定化。

  • レース後 180 / 365 / 730 日の獲得賞金
  • 重賞出走・好走
  • 後続レースの run_performance_final
  • 勝ち上がり頭数

future-aware → 学習特徴量には不採用。

11 RunPerformance 合成

run_performance_raw =
    a * time_figure
  + b * margin_adjustment
  + c * weight_adjustment
  + d * pace_adjustment
  + e * race_level_adjustment
  + f * trouble_adjustment
  • race_level_adjustment: 弱相手への 2着を減点、強相手への 2着を加点
  • trouble_adjustment: 出遅れ・不利・前壁・直線詰まり等の補正

12 追加考慮事項

不利・ロス補正

通過順・上がり・枠順から proxy を作成可能。

少頭数 / 多頭数

field_size_factor と、小頭数戦での race_uncertainty 引き上げ。

新馬・未勝利の信頼度

戦績不足時は race_uncertainty を高く、retrospective で安定化。

年齢・成長曲線

2歳・3歳春は変動が大きく、retrospective 寄せをやや強める。

距離変更・転戦

距離帯変更ペナルティ、surface switch uncertainty。

当日位置取り有利不利

day bias を使って逆風好走を加点。

16 解釈と改善ポイント

結論

run_performance 系の履歴だけで、2025 race-wise top1 的中率を 23% 前後 まで押し上げ（ランダム基準 7.66% に対して約 3.0×）。 しかも future_horizon_days = 0 の本番想定ロジックのみで達成しており、 単独で有意な予測信号として成立しています。

重要特徴量（is_win 上位）

margin_fig_avg5 perf_pct_last1 pace_fig_avg5 field_top3_gap pace_same_distance_avg5 margin_same_distance_avg5 perf_same_class_avg5 n_prev_runs perf_same_venue_avg5 perf_same_track_cond_avg5

改善サイクル要約

#改善内容状態
1distance / surface / field_size の上流補完完了
2track_variant の online 化完了
3race_level_pre_race の race 内差分化（field_relative_pre_rating / field_top3_gap）完了
4標準斤量基準への正規化と今回斤量補正完了
5条件帯標準化（_std / _pct）完了
6文脈別履歴集約（same_class / same_venue / same_track_cond / same_surface_distance）完了
7pace_same_distance_avg5 / margin_same_distance_avg5 の追加完了
8race store を source of truth 化（--race-ids）完了
9future_horizon_days = 0 を運用デフォルト化完了

頭打ち判断

改善幅は十分に小さくなり、現行スコープでは大きな改善余地は一巡。今後の改善は本ロジック単体ではなく、horse_pre_race_dataset への安全統合、pace_shape_class 別履歴集計、多段階パイプラインとの接続といった上流／下流接続の話に移行します。

18 推奨評価プロトコル

  1. 2020-2024 を履歴構築・学習用期間とする
  2. 2025 を holdout とする
  3. 2025 の指標は future_horizon_days=0 で再算出されたものを使う
  4. race_level_retrospective は holdout 予測特徴量に入れない
  5. 本番想定寄与の評価は常に online_safe 列だけで実施する

19 完全版の判断

future_horizon_days = 0 の本番想定ロジックで算出可 race_id 単位 で蓄積可 2025 holdout で 有意な予測寄与 特徴量として data/features に保持済み 再現可能な評価スクリプトを保持

結論

現時点の race performance システムは、本テーマについての 現行スコープでの完全版 とみなせる。 今後はロジック単体の改善ではなく、上流・下流接続（dataset 統合 / pace_shape 履歴 / 多段階パイプライン）に進むのが妥当。
