1. ローカルデータセット一覧

すべてのデータは BASE_DIR（プロジェクトルート）からの相対パスです。 HybridStorage 経由のカテゴリは GCS を一次ソースとし、data/cache/ をディスクキャッシュとして使用します。 それ以外はローカルファイルを直接参照します。

2. HybridStorage 経由エンドポイント (GCS + data/cache/)

_get_storage() で返す HybridStorage インスタンスを通じて storage.load(category, key) を呼ぶエンドポイントです。 GCS が一次ソースで、data/cache/{category}/{key}.json がディスクキャッシュです。 GCS 未設定・オフライン時はキャッシュのみ参照します。
エンドポイントメソッド参照カテゴリ説明
/api/data/{category}/{key} GET 任意カテゴリ race_performance (local) 生 JSON データビューア用。race_performance は data/local/features/ を直接参照
/api/race/{race_id}/bundle GET race_resultrace_shutubarace_odds race_indexrace_paddockrace_trainer_comment race_barometersmartrc_race horse_resulthorse_pedigree_5gen レース全カテゴリ + 出走馬データを並列一括取得（レースページのメイン API）
/api/horse/{horse_id}/detail GET horse_resultsmartrc_racehorse_pedigree_5gen 馬詳細情報 (過去戦績・SmartRC 指数・5世代血統)
/api/race/{race_id}/tracking-difficulty GET race_shutubarace_resulthorse_result 追走難度・ペース予想・位置取り予測
/api/person/{ptype}/{person_id}/stats GET horse_resultrace_result 騎手・調教師成績集計（horse_result の race_history を走査）
/api/race/{race_id}/predictions GET race_shutubarace_resulthorse_resulthorse_pedigree_5gen 予測結果取得（モデル推論前に storage からフィーチャを読み込み）
/api/race/{race_id}/predict POST 上記と同様 予測を強制再実行
/api/betting/pair-odds/{race_id} GET race_pair_oddsrace_shutubahorse_result 馬連オッズ詳細
/api/odds/predict/{race_id} GET race_oddsrace_shutuba オッズモデル予測
/api/odds/history/{race_id} GET race_odds オッズ履歴（スナップショット一覧）
/api/simulation/run POST race_resultrace_shutubarace_oddshorse_resulthorse_pedigree_5gen バックテストシミュレーション（期間内レースを storage から読み込み）
/api/backfill/start POST race_resultrace_shutubahorse_result… データバックフィル（GCS → ローカルへの一括保存）

6. JRA 馬場情報 (data/local/jra_baba/)

data/local/jra_baba/ ディレクトリに格納されたクッション値・含水率・馬場状態データを参照します。 定期取得は jra-baba-morning cron ジョブ（毎 10 分 05:00–09:00 JST 開催日のみ）が担当します。
エンドポイントメソッド参照ファイル説明
/api/cushion GET data/local/jra_baba/cushion_values.json クッション値・含水率レコード一覧（年・会場フィルタ可）
/api/cushion/stats GET data/local/jra_baba/cushion_values.json 年別・会場別集計統計
/api/cushion/scrape POST data/local/jra_baba/ (書き込み) クッション値を JRA 過去ページから一括スクレイプ（バックグラウンド）
/api/cushion/scrape/status GET インメモリ状態 スクレイプジョブ進捗
/api/cushion/live POST data/local/jra_baba/baba_live_{date}.json (書き込み) JRA 公式馬場ページからライブ取得（当日データ）
/api/cushion/live/status GET data/local/jra_baba/baba_poll_schedule.json ライブジョブ状態 + ポーリングスケジュール確認
/api/cushion/live/check GET data/local/jra_baba/baba_live_{date}.json 軽量更新チェック（フルスクレイプなし）
/api/cushion/schedule GET data/local/jra_baba/baba_poll_schedule.json 直近 N 日のポーリングスケジュール一覧
/api/cushion/admin/sync-gcs POST data/local/jra_baba/cushion_values.json → GCS ローカル cushion_values.json を GCS へ年別同期（開発者限定）
