Google Cloud Storage（GCS）コスト分析レポート

更新日: 2026-04-15（第3回最適化の追記）

2. キャッシュ（一時保存）の仕組み

GCSへのアクセス回数を減らすために、複数段階のキャッシュを設けています。キャッシュとは「一度取得したデータを一定時間（TTL: データの有効期限）だけ手元に保存しておく」仕組みです。

2.1 最適化前（初期状態）

L2: ディスク load() 結果 172800秒（48時間） 43200秒（12時間） 週2回以上アクセスで書き込み

6.14 prefetch_horse_timestamps() の生年範囲を拡大 — 新規

問題の内容: prefetch_horse_timestamps("2024") は馬IDの先頭4桁が "2024" の馬（2024年生まれ）しか事前取得していませんでした。実際のレースには複数の生年の馬が混在するため、カバー率が不足していました。

8. GCPコスト内訳の確認手順

実績コスト¥6,026の内訳を確認する手順:

  1. GCPコンソール → 請求先アカウントを開く

  3. 以下の各SKUの料金を確認:
    • Standard Storage - Class A Operations（list/put）
    • Standard Storage - Class B Operations（get）
    • Standard Storage（データ保存量）
    • Download Worldwide Destinations（ネットワーク下り）
