適性3D × 5世代血統（父系／母系経路の扱い）

血統適性（父・母系の遺伝的特徴がコース・距離に合うか）を精度高く評価するために、単純な「父・母父」の2段階ではなく、5代前までの血統を使ってスコアを計算します。このページでは、父方・母方の経路をどう区別して重み付けするかを解説します。

なぜ父系と母系を別々に扱うのか

遺伝の基礎知識

遺伝要素伝わり方競馬予測への注意点
常染色体 父・母からそれぞれ50%ずつ。世代が1つ上がるごとに寄与は約半分（2−g）になる research/pedigree_similarity.build_ancestor_vector と同じ計算方法
ミトコンドリアDNA（mtDNA） 母系のみが子に伝わる エネルギー代謝関連遺伝子を含むが、タイム指数との直接因果関係は学術的合意なし。過大解釈に注意
Y染色体 父系のみで受け継がれる ゲノム全体に占める割合が非常に小さく、影響は限定的

競馬・配合の慣習

概念内容本プロジェクトでの扱い
母馬・繁殖牝馬の質 産駒評価において重要視される。「母の影響」は競馬の世界で広く語られる 牝系ラインのスコアとして sire_aptitude_note に組み込み
母父（broodmare sire）の効果 血統解説や配合論でよく登場する 父1.0 + 母父0.55 というブレンド比率を採用。母方由来の影響を少し強めに設定
スコアの性質 遺伝子レベルで厳密に「父由来」「母由来」を測定したものではない 配合論・市場評価・専門家の知見を組み合わせた主観的スコアとして運用

実装上の位置づけ（重要）

  • 以下のアルゴリズムは、著者が作成したナレッジJSON（主観スコア）への重み付けルールです。遺伝子を直接測定したものではありません。
  • 再現性と比較のため、係数は src/research/note_aptitude_5gen.py にまとめており、実績データと照らし合わせながら随時調整します。

アルゴリズムの概要

  1. horse_pedigree_5gen/{horse_id}.json の ancestors（netkeiba 血統表と同順：generation×position）を読み込みます。
  2. 各祖先に 基礎重み（w_0 = 0.5g）を付与します（g は世代数）。同じ祖先が複数回現れる場合（インブリード）は重みを合算します。
  3. 血統表上の位置から 母方経路の割合（m ∈ [0,1]）を算出します。具体的には、世代 g の position を2進数で表したとき、「1の個数 ÷ g」が母方寄りの度合いを表します。
  4. 軸ごとに係数（ka）を設定し、ma = 1 + ka × (2m − 1) という式で母方寄り・父方寄りの重みを調整します（m = 0.5 で中立）。極端な値にならないようクリップ処理も行います。
  5. 祖先名を sire_aptitude_note.json の種牡馬（aliases 経由も含む）に照合し、マッチした祖先のみスコアを持ちます。軸ごとに加重平均で正規化します。
  6. 5代血統データがない馬は、従来の浅いブレンド（父・母父・牝系）にフォールバックします。

血統データが不足している場合の自動補完（レース単位）

レース適性3Dページでブレンドを「5世代」に設定すると、表示前に不足している horse_pedigree_5gen を段階的に自動取得します。すべての取得ジョブは 最高優先度（PRIORITY_URGENT_PEDIGREE_5GEN）でキューに投入され、画面上にオーバーレイで進捗が表示されます。

取得は以下の3段階で行います。

  1. 出走馬 — 出馬表に載っている各馬の5代血統を最初に取得・保存します。
  2. 第1階層 — 保存済み血統の祖先のうち、sire_aptitude_note に登録されている種牡馬をレース内でユニーク化し、不足分だけ取得します。
  3. 第2階層 — 第1階層で取得した各馬の血統から、さらにナレッジ種牡馬IDを収集し、不足分だけ取得します。段階を分けることで、一度に大量のジョブが投入されるのを防いでいます。
  • ブレンド計算は引き続き 各出走馬1件のJSONファイル内の祖先名のみを参照します。
  • UIは複数ラウンド（各段完了後に再POSTしてポーリング）で、全段完了まで繰り返します。
