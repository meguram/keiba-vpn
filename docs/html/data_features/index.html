馬場傾向×血統（preprocess・track_bias_pedigree）

race_result の芝/ダ/障（surface）と場・日付・発走時刻・馬場状態・距離で、 傾向は 日×場×芝ダ障×馬場3区分×距離4帯（欠損時は場×芝ダ障のみ）で集計。 当日は当該レースより前に終了した同一ストラタの勝者上がり3F平均、 前日は同一場・芝ダ障の粗いキーでの前日平均。 父・母父は race_shutuba 由来で芝/ダ別の当該レースより過去のみの累計勝率。 交互作用列は勝率 ×（当日傾向 − 前日傾向）に加え、 当日/前日ブレンドに馬場・距離の乗算重みを掛けた合成 track_bias_winner_3f_weighted（既定 w_same=0.75, w_prev=0.25、馬場・距離は 1.0）と、 勝率 × その合成の相性度（sire_x_track_bias_weighted 等）。 research/tune_track_bias_weights は version 2 JSON（cond_multipliers / dist_multipliers）を出力可能。 メタ JSON に実効重み・乗算（指定時）が記録される。チューニング前は LayerA を再ビルド（track_bias_cond_bucket 等が必須）。

track_bias_winner_3f_same_day_prior, track_bias_winner_3f_prev_day, track_bias_winner_3f_diff, track_bias_winner_3f_weighted, track_bias_cond_bucket, track_bias_dist_band, track_bias_stage_key, sire_prior_win_rate_surface, dam_sire_prior_win_rate_surface, sire_x_track_bias_diff, dam_sire_x_track_bias_diff, sire_x_track_bias_weighted, dam_sire_x_track_bias_weighted

4. 明示的に入れないもの（現段階）

  • 当日の 単勝オッズ・人気（race_shutuba / race_index 由来）
  • feature_builder.PUBLIC_INDICATOR_SET に含まれる市場系列（結合後に一括ドロップ）
  • 前走生の prev*i*_odds / prev*i*_popularity（派生特徴に変換する前段の flat には、馬柱 JSON 内に含まれる場合あり → 現ビルドは生列としては展開しない）
  • ホワイトリストで明示除外のテーブル（特徴としては未結合）: race_result_lap, race_barometer, smartrc_race 等
  • 指数コメント全文 all_txt_c（数値化前のためベースライン Parquet から除外）

5. 学習パイプラインでの読み込み順

  1. data/modeling/layer_a_train.parquet が存在すれば 最優先（pipeline/trainer.py の _load_or_build）
  2. 無ければ FeatureStore スナップショット → 列ストア → CSV → GCS フォールバック
  3. 分割: data/meta/modeling/dataset_split_manifest.json の model_selection_primary（2020–23 train / 2024 valid / 2025 test）
  4. 記録: MLflow（python3 -m pipeline.run_baseline_train）

6. 当日予測パイプラインとの差（注意）

RaceDayPipeline は build_race_features() 由来で列数が多い場合があります。 本 Parquet は flat テーブル直接結合の BaselineV1 最小セットです。本番当日と完全に同じ列に揃えるには、今後の段階で派生特徴の統一が必要です。
