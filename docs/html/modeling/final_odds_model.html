MODELING · FINAL ODDS

最終オッズ予測モデル

発走前の出馬表・調教・速度指数・過去走データから単勝・複勝オッズを推定する LightGBM 3 ヘッドモデルの設計・学習・推論フローをまとめたページです。 src/pipeline/models/final_odds_*.py の実装に基づきます。

概要

最終オッズ予測（final_odds）は MLflow モデルカタログに登録された アクティブなモデルです。発走直前に「市場が形成するオッズを事前に推定する」という 目的で設計されており、LayerB（市場統合）の検証や safe_ev 計算の補助として使います。

設計上の注意

このモデルは「市場が付けるオッズを予測する」ものです。 LayerA の着順確率モデルとは独立して扱い、両者を直接比較して edge を求める設計です。 予測オッズに当日の実際のオッズを混ぜた特徴量は使いません（情報リーク防止）。

3 ヘッド構造

win
単勝オッズ
exp(pred) = win_odds
place_min
複勝オッズ（下限）
exp(pred) = place_min
place_max
複勝オッズ（上限）
exp(pred) = place_max

各ヘッドは対数変換したオッズ（target_log_win_odds 等）を回帰ターゲットとして個別に学習します。 推論時は exp(pred) で元のオッズスケールに戻します。最小値は 1.0 にクリップします。

リーク防止

_filter_horse_history_by_date() がレース日より後の過去走履歴をフィルタアウトします。 ターゲット（確定オッズ）もレース後に確定する値のため、学習データは race_result の確定値を使い、推論時は race_shutuba のみで動作します。

特徴量エンジニアリング — enrich_final_odds_features()

build_race_features() の出力に対して、以下の特徴量を追加します。

フィールド相対統計（レース内比較）

以下の数値列それぞれについてレース全馬との比較指標を計算します。

speed_max speed_avg speed_best_dist avg_finish_5 career_win_rate same_surface_win_rate paddock_score ... 計25列
派生列のパターン内容
fld_z_{col} レース内 z-score（平均0・標準偏差1に正規化）
fld_rank_{col} レース内順位（上位ほど小さい値、0〜1 に正規化）
fld_gap_best_{col} レース内最高値との差（0 = 最高値）

追加エンコーディング列

列名内容
horse_number_norm 馬番 / 出走頭数（0〜1 に正規化）
bracket_gate_ratio 枠番 / 8
surface_code 1=芝, 2=ダート, 0=その他
direction_code 1=右, 2=左, 0=その他
track_condition_code 1=良, 2=稍重, 3=重, 4=不良
venue_code 開催場コード（1〜10: 札幌〜小倉）
grade_code グレードコード（1=G1, 2=G2, ... 10=新馬, 11=障害）
distance_km 距離 (m) / 1000
is_turf / is_dirt 芝フラグ / ダートフラグ
log_field_size log(1 + 出走頭数)
field_speed_std / field_speed_spread レース内 speed_max の標準偏差・最大幅（混戦度指標）

モデルロード順序（FinalOddsPredictor）

FinalOddsPredictor.load()
  1. models/final_odds_bundle.json  が存在すれば bundle から読み込み
     └─ heads.win / heads.place_min / heads.place_max それぞれの model_txt を lgb.Booster に変換
  2. なければ MLflow Registry から artifacts をダウンロード
     └─ model_win / model_place_min / model_place_max フォルダを download_artifacts()
  3. どちらも失敗した場合は OddsPredictor（ルールベース）にフォールバック

bundle.json の構造

{
  "feature_cols": ["horse_number_norm", "fld_z_speed_max", ...],
  "heads": {
    "win":       { "model_txt": "... LightGBM テキスト形式 ..." },
    "place_min": { "model_txt": "..." },
    "place_max": { "model_txt": "..." }
  }
}

推論フロー

predict_final_odds(features_df)
  │
  ├─ FinalOddsPredictor.load()  →  bundle / MLflow / フォールバック
  │
  ├─ enrich_final_odds_features(features_df)
  │    └─ フィールド相対統計 + エンコーディング列を付与
  │
  ├─ feature_cols を bundle.json から取得（なければ select_model_feature_columns()）
  │    └─ 欠損列は 0.0 で補完
  │
  ├─ Booster.predict(X) × 3 ヘッド
  │
  └─ exp(pred) でオッズスケールに変換
       ├─ win_odds  = max(exp(win_pred), 1.0)
       ├─ place_min = max(exp(place_min_pred), 1.0)
       └─ place_max = max(exp(place_max_pred), place_min + 0.1)

FinalOddsTrainer の動作

FinalOddsTrainer.train()
  ├─ build_final_odds_dataset(storage, train_years, eval_years)
  │    ├─ race_shutuba / race_index / race_oikiri / horse_result を並行ロード
  │    ├─ enrich_final_odds_features() で特徴量拡張
  │    └─ target = log(確定オッズ) を付与
  │
  ├─ 3 ヘッドを順番に lightgbm.train() で学習
  │
  ├─ MLflow Run を start_training_run() で開始
  │    └─ params / metrics / artifacts を自動ログ
  │
  └─ models/final_odds_bundle.json に保存 + Registry 登録
