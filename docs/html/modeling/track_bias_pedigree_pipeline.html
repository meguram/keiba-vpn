血統×コースバイアス（track_bias_pedigree）パイプライン

目的

コースバイアス（コース形状や馬場状態が特定の脚質を有利にすること）は、血統適性と組み合わせることで予測精度が向上します。このパイプラインは、当日・前日の馬場傾向（芝/ダ・競馬場・発走順）と出馬表の父・母父の芝/ダ別累計勝率を組み合わせたリークなしの特徴量を LayerA に付与します。

なお、5世代血統グラフや GCS の horse_pedigree_5gen はこのパイプラインの必須入力ではありません（詳細は後述）。

データフロー（GCS 負荷を抑える設計）

段階 データ源 アクセス
特徴計算 data/local/tables/<年>/race_result_flat.parquet ローカルのみ
血統名 race_shutuba_flat の sire / dam_sire ローカルのみ
母表 src/pipeline/layer_a_dataset.py → data/modeling/layer_a_train.parquet ローカル
  • 学習・チューニング・評価はすべて上記の Parquet ファイルを読むだけで完結します。推論時も同様に、当日以前の race_result と出馬表から同じ計算式で再現できます。
  • 5世代血統（HTML/GCS）を使うのは src/research/pedigree_similarity.py など別系統であり、馬場傾向の特徴量計算には不要です。必要な場合は一度だけバッチ取得し、data/local/ またはローカルキャッシュに保存します。以降は再取得しない方針を推奨します。

重みチューニング（当日 vs 前日 + 馬場区分 + 距離帯）

加重合成のロジックは src/pipeline/track_bias_pedigree.py で定義されています。主なパラメータは以下の通りです。

パラメータ既定値 / 区分説明
当日/前日ブレンドw_same=0.75 / w_prev=0.25w_same × 当日（該当レース前）+ w_prev × 前日（合計 1）
馬場状態3区分good / yielding_soft / heavy_bad良 → good、稍重・重 → yielding_soft、不良 → heavy_bad。乗算重みを掛けグループ内平均 1 に正規化。
距離帯4区分スプリント / マイル / ミドル / ロング≤1400m / ≤1800m / ≤2200m / それ以上
傾向集計キー（細）日×場×芝ダ障×馬場3区分×距離4帯データ不足時は粗いキーにフォールバック
傾向集計キー（粗）日×場×芝ダ障フォールバック先
チューニング出力 JSON バージョンversion 2best_same_day_weight / best_prev_day_weight + cond_multipliers / dist_multipliers を含む。version 1 も load_track_bias_weights_config で読み込み可。

チューニングの実行手順:

  1. 馬場区分・距離帯の列を含めて LayerA を再ビルドします（古い Parquet には列がないため）。
    python3 -m src.pipeline.build_layer_a_dataset
  2. Optuna で valid の NDCG@3 を最大化します（GCS への接続不要）。
    python3 -m src.research.race.tune_track_bias_weights --n-trials 60
  3. 出力した重みファイルを使ってデータセットを再ビルドします。
    python3 -m src.pipeline.build_layer_a_dataset --track-bias-weights-json data/meta/modeling/track_bias_weight_best.json
  4. 血統×馬場の特徴量のみで単体検証する場合（任意）。
    python3 -m src.research.race.evaluate_track_bias_pedigree_only --weights-json data/meta/modeling/track_bias_weight_best.json

test 年のメトリクスを JSON に含めたい場合は、--years にテスト年（例：2025）を含めてビルドしてください。train+valid のみの Parquet では test はスキップされます。

チューニングの最適化はvalid のみを対象とし、test は最終報告用に残します（リーク防止）。

さらに精度を高めるための方針（優先度順）

優先度方針説明
1重みの最適化当日/前日・馬場3区分・距離4帯の係数をデータで調整（上記の手順）。まずここから始めるのが効果的。
25世代血統の活用父・母父名の単純な一致を超えた類似度が必要な場合のみ、ローカルキャッシュ済みの血統グラフを別特徴として結合。このモジュールとは独立して管理し、GCS をリアルタイムのパスに置かない。
