レース直前 — 使用可能リスト（ホワイトリスト）

目的

このドキュメントは、発走 10 分前の時点で取得できる情報だけを使って、勝率・連対率・複勝率 モデルを構築するための入力列使用可能リスト（ホワイトリスト）を定義します。

設計は以下の 2 層に分けます。

  • LayerA — 市場から独立した実力・状態推定モデル。単勝/複勝/人気/連系オッズ は含めません。
  • LayerB — 市場統合・馬券最適化レイヤ。LayerA の馬単位確率に加えて、直前オッズ・連系オッズを加えます。

共通ルール

採用原則

  • 基準テーブルは race_shutuba_flat
  • 主キーは race_id + horse_number
  • レース結果が確定してからでないと得られない情報は使わない
  • 疎なデータ（欠損が多い列）は捨てず、presence_flag（有無フラグ）とセットで保持する
  • 生テキストはそのまま学習に使わず、集約・辞書化・スコア化して数値に変換する

明示的に除外するテーブル（レース結果を含むため）

  • race_result_flat
  • race_result_lap_flat
  • race_barometer_flat
  • smartrc_race_flat

注意事項

  • race_shutuba_flat と race_index_flat に含まれる odds / popularity 列は LayerA では使いません
  • オッズは必ず race_odds_flat と race_pair_odds_flat のみから取得します
  • smartrc_race の runners 列は前情報化できる可能性がありますが、現時点では結果混入リスクが高いため初期データセットから除外します

ベースライン優先原則

参考資料では、最初に「指数中心のシンプルなベースライン」を作り、そこから特徴量（予測に使う数値）を段階的に追加していく戦略が有効でした。本プロジェクトでも、最初の比較対象を以下の 2 つに固定します。

  • BaselineV1
    • race_shutuba_flat
    • race_shutuba_past_flat
    • race_index_flat
    • horse bundle 由来の戦績・血統集約
  • ExtendedV1
    • BaselineV1 のすべて
    • race_oikiri_flat の集約
    • race_paddock_flat
    • race_trainer_comment_flat
    • クッション値・開催文脈の強化特徴

この 2 つを常に同時出力し、追加特徴量の効果を差分で評価します。

9. GCS 由来の馬情報バンドル

feature_builder が参照する horse_profile, race_history, pedigree, myostatin, aptitude 系の情報は LayerA の中核として採用します。

LayerA で採用する主な派生情報

情報種別内容
戦績集計全キャリアの着順・勝率・複勝率
直近走の推移直近走の着順・上がり・時計トレンド
距離帯適性距離帯別の勝率・好走率
芝/ダート適性芝・ダート別の成績
コース相性競馬場別の成績
血統適性父・母父の芝ダ別実績由来スコア
ミオスタチン由来の距離適性 proxy遺伝子情報に基づく距離適性の代理変数

運用ルール

  • レース後の結果をそのまま連結しない
  • 対象レース日以前の履歴だけで集計する
  • 集計基準日（いつ時点の情報か）を必ず明記する

LayerA 最終採用対象

データ源採用内容
race_shutuba_flat非市場列（オッズ・人気を除く全列）
race_shutuba_past_flat履歴集約特徴量
race_index_flat指数列・派生列
race_oikiri_flat馬単位集約特徴量
race_paddock_flatスコア化列
race_trainer_comment_flatスコア化列
GCS horse bundle履歴・血統・適性集約
JRA 馬場情報クッション値・含水率・proxy 系

LayerA の追加ルール

  • odds を直接使わない場合でも、オッズと強く相関するリーキーな派生列を後から混入しない
  • カテゴリ encoding は必ず train fold 内で作成する
  • ベースライン段階では、生テキスト埋め込みや大型の外部特徴量は含めない

LayerB の最終採用対象

データ源採用内容
LayerA 出力最終馬単位確率
race_odds_flat単勝・複勝オッズ・人気
race_pair_odds_flat連系馬券オッズ
市場派生差分特徴量（対人気・対中央値等）

LayerB の追加ルール

  • 市場情報（オッズ）を使うのは確率推定モデルではなく意思決定モデルの側と明確に区別する
  • 高オッズ帯の期待値は確率誤差に弱いため、安全域付き期待値を前提にして扱う
  • 最終オッズを使った評価は simulation_only（シミュレーション専用）として管理する

学習前チェックリスト

  • race_result, race_result_lap, race_barometer, smartrc_race が混入していないか確認する
  • odds / popularity が LayerA に含まれていないか確認する
  • race_oikiri_flat を 1頭1行に集約済みか確認する
  • 疎なテーブルに presence_flag（有無フラグ）を付与済みか確認する
  • 未来日付の履歴が混在していないか確認する
  • データ分割（split）前に target encoding を作っていないか確認する
  • BaselineV1 と ExtendedV1 の両方を比較できる形で保存済みか確認する
