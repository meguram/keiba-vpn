Ch1 · システム設計

MLflow（モデルの保管・バージョン管理ツール）モデリング・予測プラットフォーム

競馬予測の学習・Registry・モデル実行サービス・API キャッシュを モデルキー単位 で統一する基盤です。

アーキテクチャ

# 学習フロー pipeline/models/*.py → MLflow Tracking + Model Registry (catalog.ModelSpec) → 任意: mlflow models serve (Docker :5001–:5010) # 予測実行フロー (API / バッチ) → モデル実行サービス (/invocations) … あれば優先 → ローカル Booster (Registry / mlflow/runs) → ヒューリスティック / ルール # API 高速化 → HybridStorage キャッシュ (InferenceCacheMixin) → 事前計算 CLI (scripts/maintenance/precompute_*.py)

モデルカタログ

正の定義: src/pipeline/mlflow/catalog.py の MODEL_CATALOG

model_key 用途 Registry 名 Serving ポート lifecycle
keiba_lgbm 既存着順 GBDT（木を積み上げた機械学習モデル） keiba-lgbm 5010 active
tracking_difficulty 追走難度・位置取り tracking-difficulty-lgbm 5001 active
finish_order 次期着順予測 keiba-finish-order-lgbm 5002 planned
final_odds 最終オッズ予測 keiba-final-odds-lgbm 5003 planned
pace_predictor 1F/3F ペース pace-predictor-lgbm 5004 active

設定の上書き: config/settings.yaml → mlflow.models.<key>
環境変数: KEIBA_MLFLOW_SERVE_<KEY>_URI（例: KEIBA_MLFLOW_SERVE_FINISH_ORDER_URI）

新しい予測タスクを追加する手順

  1. catalog — ModelSpec を MODEL_CATALOG に追加する
  2. settings — mlflow.models.<key>.serve_uri / serve_port を設定する
  3. storage — CATEGORY_MAP に cache_category を追加する（race など）
  4. 学習 — train_utils.log_lightgbm_and_register("your_key", model, ...) を呼び出す
  5. 予測処理 — inference/<task>_service.py を作成する（InferenceCacheMixin + get_or_compute を継承）
  6. API — 薄い GET エンドポイントと、任意で POST precompute を追加する
  7. Docker — mlflow/server/docker-compose.yml に serve サービスを追加する（profile 対応可）

コード例

学習

from src.pipeline.mlflow.train_utils import start_training_run, log_lightgbm_and_register

with start_training_run("finish_order", params={"n_estimators": 500}):
    log_lightgbm_and_register(
        "finish_order",
        booster,
        metrics={"auc": 0.72},
        feature_names=cols,
    )

予測実行（LGBM バッチ）

from src.pipeline.mlflow.runtime import get_serve_client, load_lightgbm_booster, booster_feature_names

cols   = booster_feature_names("finish_order") or FEATURE_COLUMNS
client = get_serve_client("finish_order")
if client and client.is_available():
    scores = client.predict_dataframe(cols, matrix)
else:
    model = load_lightgbm_booster("finish_order")

運用コマンド

目的コマンド
全モデルのヘルスチェック GET /api/inference/health
CLI からヘルスチェック python -m src.scripts.maintenance.mlflow_platform_health
Tracking サーバーを起動 docker compose -f mlflow/server/docker-compose.yml up -d mlflow
追走難度のモデル実行サービスを起動 docker compose -f mlflow/server/docker-compose.yml up -d mlflow-serve-tracking
着順・オッズのモデル実行サービスを起動 docker compose -f mlflow/server/docker-compose.yml --profile finish-order up -d

ローカル開発で Registry の学習済みファイルが mlruns/ を参照している場合は、リポジトリルートに mlruns → mlflow/runs のシンボリックリンクを配置してください（ensure_mlruns_symlink() が自動作成します）。
