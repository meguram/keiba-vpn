1. 本プロジェクト（keiba-vpn）の現状のデータ形態

scraper/storage.py の HybridStorage は、原則 JSON オブジェクトを GCS に保存（category + key）。race_lists など一部は ローカル 扱い。カテゴリ例:

  • レース系: race_shutuba, race_result, race_odds, smartrc_race, race_result_lap など
  • 馬系: horse_result, horse_training, horse_pedigree_5gen など
  • その他: 予測 race_predictions など

API・モニタは 多キーの読み出し（storage.load(category, id) や並列 GET）、欠損検知・学習特徴は data/meta や pipeline / ローカル Parquet 層（scraper/local_tables.py 等）と併用される構成が想定される。

4. 価格の考え方（本ページの試算式）

Google Cloud Storage（単一リージョン・Standard）

公示（東京を含む単一リージョン行）: $0.000027397 / GiB 時（Storage pricing | Google Cloud）。

月あたり GiB 課金（730 時間/月 近似）:
単価 ≈ 0.000027397 × 730 ≈ 0.020 $/GiB/月。

試算: 500 GiB × 0.020 ≈ 10.0 $/月（保存のみ、オーダー）。
操作: Class A/B 課金は要別計算（本シナリオ 30 万 GET/月 でも多くの場合 保存額の次）。同一リージョン内 Egress 無料帯の条件等は要確認。

Cloud SQL (PostgreSQL 等) — 小規模本番の「床」

CPU/メモリ/ディスク 常時起動分が支配的。1 vCPU 級 + 小ディスクで ~50–200 $/月 帯（リージョン・版で変動、前回会話のレンジと同程度）。10k reads/日はインスタンス床に比べ小さい。

BigQuery（オンデマンド分析・公式抜粋）

一次情報: BigQuery pricing | Google Cloud — オンデマンドは 処理したバイト（TiB）あたり。公示例: $6.25 / 1 TiB（クエリ）/ 月 / アカウント。さらに 毎月最初の 1 TiB のクエリ処理は無料（同ページ「On-demand (per TiB)」節）。

試算: 月に 2 TiB スキャン の分析が 1 回だけの場合、無償 1 TiB 控除後 (2 − 1) × $6.25 ≈ $6.25/月。無償帯を使い切らない小規模なら $0 に近い月もある。列・パーティション最適化でスキャンを抑えるのが定石。ストレージ料は同ページの「Storage pricing」参照（物理容量・ロングターム等）。

8. 予測 AI 観点の扱い

  • 学習・大規模特徴量再計算 — 行指向 RDB より 列存 Parquet/外部テーブル + 列指向 DW/Spark の方がコスパ・帯域が出やすい。本リポの export_tables 系の思想と整合しやすい。
  • オンライン推論 — 低遅延の キャッシュ（Redis/メモリ）+ OLTP/キー価格ストア か、事前マテリアライズ された少数列。
  • 中間合意 — 正本は オブジェクト（GCS/S3）、メタ/索引は SQL/検索、学習は DW/バッチ の 三段 が扱いやすい。
