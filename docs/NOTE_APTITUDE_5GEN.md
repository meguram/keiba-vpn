# 適性3D × 5世代血統（父系／母系経路の扱い）

## 調査メモ（なぜ父／母で分けるか）

### 生物学・遺伝

- **常染色体**は父・母から各50%ずつ受け取るのが基本。5世代前の特定祖先からの**期待寄与**は、簡易モデルでは経路長 \(g\) に対し **\(2^{-g}\)**（各世代で半減）で表現できる（本リポジトリの `research/pedigree_similarity.build_ancestor_vector` と同じ発想）。
- **ミトコンドリアDNA（mtDNA）は母系のみ**が子に伝わる。エネルギー代謝関連の遺伝子を含み、種によっては運動能力との議論があるが、**競走馬のタイム指数との因果はハッキリした合意はない**（過大解釈に注意）。
- **Y染色体**は父系男子のみで伝播するが、ゲノム全体に占める割合はごく小さい。

### 競馬・ブリーディングの慣行

- **母馬・繁殖牝馬の質**は産駒評価で重視される（「母の影響」が語られる）。
- **母父（broodmare sire）効果**は血統解説・配合論で頻出で、本プロジェクトの `sire_aptitude_note` でも **父1.0 + 母父0.55** のブレンドが既に「母方由来を少し強める」ヒューリスティックになっている。
- これらは **統計的に厳密に分離された「父由来／母由来遺伝子」の計測ではなく**、配合・市場・主観ナレッジの混合に近い。

### 実装での位置づけ

- 以下は **著者ナレッジ JSON（主観スコア）への重み付けルール**であり、**遺伝子レベルの実測ではない**。
- **再現性・比較のため**、係数は `research/note_aptitude_5gen.py` に集約し、検証しながら調整する想定。

## アルゴリズム概要

1. `horse_pedigree_5gen/{horse_id}.json` の `ancestors`（netkeiba 血統表と同順：`generation`×`position`）を読む。
2. 各祖先に **基礎重み** \(w_0 = 0.5^{g}\) を付与（`g = generation`）。同名複数回はインブリードとして重みが足し上がる。
3. 各祖先の血統位置から **母方経路の割合** \(m \in [0,1]\) を定義：世代 \(g\) の `position` の下位 \(g\) bit における **1 の個数 / g**（母方へ進んだ世代数の割合）。
4. 軸ごとに係数 \(k_a\) を持ち、  
   \(m_a = 1 + k_a \cdot (2m - 1)\)  
   で **母方寄り経路を強める／父方寄り経路を強める** を調整（\(m=0.5\) で中立）。\(m_a\) はクリップして暴れを防ぐ。
5. 祖先名を `sire_aptitude_note.json` の種牡馬（＋`aliases`）に解決し、**解決できた祖先だけ**がスコアを持つ。軸ごとに  
   \(\text{blended}[a] = \sum_i w_i m_{a,i} v_{a,i} / \sum_i w_i m_{a,i}\)  
   で正規化。
6. 5世代データが無い馬は **従来の浅いブレンド**（父・母父・任意牝系）にフォールバック。

## 欠損時の自動取得（レース単位）

- **レース適性3D**ページでブレンドを「5世代」にしたとき、表示前に `horse_pedigree_5gen` を **段階的** に揃える。いずれも **最優先優先度**（`PRIORITY_URGENT_PEDIGREE_5GEN`）でキュー投入し、画面上にオーバーレイで進捗を表示する。
  1. **出走馬** — 出馬表の各馬の 5 世代血統を先に取得・保存する。
  2. **第1階層** — 保存済み血統の `ancestors` のうち、名前が `sire_aptitude_note` に解決できる種牡馬の `horse_id` をレース内でユニーク化し、欠損分だけ取得する。
  3. **第2階層** — 第1階層の各頭の血統から同様にナレッジ種牡馬 ID を集め、欠損分だけ取得する。  
     （理論上は馬あたり多くのスロットがあるが、ナレッジに載る名のみ・レース内ユニークに抑え、かつ段を分けることで一括過多投入を避ける。）
- ブレンド本体は引き続き **各出走馬の 1 件の JSON** 内の祖先名だけを参照する。
- UI は **複数ラウンド**（各段のキュー完了後に再 POST → ポーリング）で全段完了まで繰り返す。
- API 単体利用:
  - `POST /api/pedigree/race-ensure-5gen` — body: `{ "race_id": "..." }`
  - `GET /api/pedigree/race-ensure-5gen/status?session_id=...` — 投入セッションの完了状況
- 実装: `research/race_pedigree_5gen_prefetch.py`、キュー優先度は `scraper/job_queue.py`。
